It's Time to Rethink Programming Languages from the Ground Up. As noted by John Backus 24 years ago, programming languages are too closely tied to the von Neumann architecture, and we are way overdue for a change. [kuro5hin.org]

It's true that we need to start using languages that are better suited for what we do, certainly when programming for the web. But there are a lot of languages out there, which hardly no one uses.

Having made an XML-RPC client, I also needed an XML-RPC server. So I made one, and I got it to validate on Userlands's XML-RPC Validator. Here is the validator1 library that get's called by the XML-RPC server (just a plain text file). As you can see they are plain PHP functions, which should make it very easy to create new services.
