Javascript and Frames

Two days ago I wanted to link to an article on Joeri's weblog. This was a problem, because Joeri's site uses frames, and I didn't want to link to a page without the other frames around it. As he didn't have any server side scripting options, we had to solve it with javascript. The final solution: when you enter the site, an intermediate page is loaded in the main frame. This page only contains a script element with this code: location.replace(unescape(top.location.search.substr(1))||'blogger.html'). This loads the page given after the question mark, or blogger.html by default. Now I can link to any page by adding a question mark to "http://joeri.mulder.com/", with the escaped uri of the page after that. So http://www.joeri.mulder.com/2001_06_10_archive.html#4102444 becomes http://www.joeri.mulder.com/?2001_06_10_archive.html%234102444
