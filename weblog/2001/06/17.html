Scoble has the scoop on how Smart Tags may be pared down considerably in future MSIE builds. [Hack the Planet]

Good. With all the uglyness gone, Smart Tags is quite a nice feature, although it's not a standard. First of all, if you'd add Smart Tags to your site, it doesn't hurt the users that don't use Internet Explorer. Secondly it adds semantics to the webpage. The implementation doesn't do just text matching, but it requires you to add an element, with a namespace, around the word that you want to be tagged. F.e. the Dublin Core could make a Smart Tag handler available for elements like dc:source. If people would start adding these elements to their texts, search engine could provide extra search options: "search for webpages that have used w3future.com as a source".
