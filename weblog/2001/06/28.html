Scripting and Data

Lisp is an intriguing language, in that the code and the data have the same form, a tree. This leads to interesting possibilities because the code can be manipulated like data. So when dreaming about a perfect scripting language, which would have an attributed tree as it's basic data type, I imagine the code to be an attributed tree as well.

On a different thread I've also thought a lot about data formats. In formats like HTML and CSS it is often useful to have values that depend on other values. In IE5 you can actually have expressions in attributes. In an ideal general purpose data format it would be possible to insert expressions anywhere which could even evalutate to complete subtrees.

Then I realised that my ideal scripting language and my ideal data format would be the same thing. Then what would this format look like? It could look like anything: Lisp, CSS, XSL. It's not really that important. There are so many ways to serialize attributed trees. And how would a user interface look like to create such programs/data files? I would like to see something like Excel, but instead of having the cells aranged in a table, they would have to be arranged in a tree.

The IE team has kept it's promise, and in build 2501 the "Always have Web pages display available smart tags" option is gone. But who cares:
