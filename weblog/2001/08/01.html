 Wednesday, August 01, 2001

In July my RSS file has been downloaded 15741 times. That's 134Mbyte, 55% percent of my total traffic. This is way to much, if you compare that to the 2617 times my html weblog has been downloaded last month. This looks like a scalability problem. But I have a monthly traffic limit of 1500MByte, so I don't worry.

More two-way inspiration: Someone at Northern Magic has created an XML-RPC spell checker similar to mine. He or she uses the built-in XML components (MS or Mozilla), to directly call the XML-RPC service.

Rick has used JSRS to create a search page for his weblog. Be sure to check out his source to see that the magic is completely hidden. If Curl is a revolution, then this sure is. It's time to think about web applications as simple solutions, not as complex toys.

Standard: "Curl just might revolutionize the way Web sites are made. Who thinks so? Tim Berners-Lee." [Scripting News]

Curl has some obvious shortcommings. It's not really HTML, so a lot of things I expect don't work, like my scrollwheel or context menus. Also they think they are important enough to have a traybar icon and stay in my memory. If you don't want to scare your website visitors, stick with Javascript.
