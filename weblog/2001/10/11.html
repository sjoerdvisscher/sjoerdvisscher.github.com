A new version of Beyond JS is available. We have (Dan and I) split the library. beyond.js now contains platform unspecific code only. It can be used in the browser, but in ASP or WScript too. beyondBrowser.js contains browser specific code, and beyondXML.js contains XML specific code.

An example of what can be done with beyond.js. First an example of some Haskell code, which is a used to explain some of the benefits of Haskell:

qsort []     = []
qsort (x:xs) = qsort elts_lt_x ++ [x] ++ qsort elts_greq_x
                 where
                   elts_lt_x   = [y | y <- xs, y < x]
                   elts_greq_x = [y | y <- xs, y >= x]

With beyond.js, this type of code can be done in javascript too. This is the code to add a qsort method to arrays:

_AP.qsort = _AP.recurse.curry(function(head, tail) {
	var elts_greq = [];
	var elts_lt = tail.filter(function(v) { return v < head; }, elts_greq);
	return elts_lt.qsort().concat(head, elts_greq.qsort());
});

// this is how to use qsort:
alert([ 5, 3, 4, 2, 1 ].qsort());

_AP is a shortcut for the Array prototype. 'recurse' is a method of arrays to help program recursion. 'curry' is a method of functions that returns a new function, that works exactly like the original function but with some arguments prefilled. Finally, 'filter' is a method of arrays, that returns all items that satisfy the condition.
