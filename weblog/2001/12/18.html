RDF/XML Syntax Specification (Revised)
The W3C RDF Working Group is trying to explain RDF again. They try to use the Infoset, but it looks like someone had fun making it a bit more complicated than needed. First they translate the Infoset into a new SAX-like model, choosing incedibly weird terms. An end element node??? Hello, this isn't the 90's anymore!! At least call them end tag nodes. The term node isn't very smart either, because it's a completely different thing than the nodes defined by the DOM.

Then they use this new model to build the RDF/XML Grammar. To make things a little easier they introduce another grammar notation. It looks like the notation used for constuctors in the XQuery 1.0 Data Model. And they could have used large portions of that definition. Yet the W3C wouldn't be the W3C if it wasn't completely different. *Sigh*

Finally for each production in the grammar they explain how they add to the real RDF model... Using yet another syntax. (N-Triples)

So, if you want to parse RDF, you have to takes these steps: 1. Read the XML syntax 2. Transform it to the Infoset 3. Transform the Infoset into a list of nodes 4. Transform (again?) the list of nodes to RDF triples. And these triples actually form a labeled directed graph (which should be aparent from the original syntax??) To make it more easy: the transformations in step 2 and 3 are only defined in English, and the transformation in step 4 is described in a proprietary grammar. Good luck!
