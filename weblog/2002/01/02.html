 Wednesday, January 02, 2002

Edd Dumbill: Patent threat to W3C's RDF technology. [Scripting News]

How serious is this. And how can it be that the company that has the intellectual property does not have a website??


The first new verb for 2002, file.synchTableWithFolder, is a dandy. We couldn't do it earlier because the CPUs weren't fast enough. Now we can do callbacks, macros, renderers, etc that live in the file system, not the object database; and make upgrades easier. Should be fun. (It's not released yet, but will be shortly. Discuss on the Frontier-server list.) [Scripting News]

Dave Winer and his team really got something interesting here: synchronisation. OPML is going to get it too. Querying is old fashioned, from the time only a few needed to view the data. With webservers, every visitor wants realtime data, so synchronisation is much better. But we have to work with a lousy infrastructure for realtime data. Automatically sending notifications that a file has changed is nice, but it would be nicer to receive the changes, which can update a copy of the file automatically. And if I'm transforming an XML file with XSLT, and the source XML file changes, I want to see the result change too, in realtime. Nevertheless, Userland is going to have something running shortly, which is a nice start.
